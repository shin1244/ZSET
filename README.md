# Structures

C++로 직접 구현한 자료구조 모음. 개념 이해 → 구현 → 스트레스 테스트 검증의 순서로,
게임 서버 개발에 필요한 자료구조를 하나씩 쌓아가는 저장소입니다.

## 구성

```
Structures.sln
├─ common/
│   └─ DoubleLinkedList.h
├─ ZSET/ 
│   ├─ SkipList.h
│   └─ ZSet.h
└─ TimingWheel/ 
    └─ TimingWheel.h
```

각 자료구조는 독립 프로젝트로 분리해, 개별 스트레스 테스트를 따로 실행할 수 있도록 구성했습니다.
공용으로 쓰이는 양방향 연결 리스트는 `common`에 헤더 온리로 두고 include 경로로 재사용합니다.

## ZSet

Redis의 Sorted Set을 참고한 정렬 집합입니다. **점수(score) 기준 정렬**과
**멤버 기준 조회**를 모두 빠르게 지원하는 것이 목표입니다.

### 설계

`SkipList` + `unordered_map`의 복합 구조로 구현했습니다.

- **SkipList** — 점수 순 정렬을 담당. 범위 조회, 순위(rank) 조회를 로그 시간에 처리합니다.
- **unordered_map** — 멤버 → 점수 매핑을 담당. 특정 멤버의 점수·존재 여부를 O(1)로 조회합니다.

두 구조가 같은 데이터를 다른 관점으로 색인하기 때문에, "점수로 정렬된 순위"와
"멤버로 직접 접근"이라는 상반된 요구를 동시에 만족시킬 수 있습니다.
그 위에 사용자 정보를 관리하는 서비스 계층을 얹어 랭킹 기능으로 사용합니다.

## TimingWheel

시계처럼 원형으로 배열된 슬롯을 커서가 한 틱씩 이동하며,
각 슬롯에 걸린 타이머를 만료 처리하는 스케줄러입니다.

### 설계

- 슬롯 배열(`WHEEL_SIZE`개)의 각 칸은 양방향 연결 리스트(`common`의 `DoubleLinkedList`)로 구성됩니다.
- 삽입 시 `만료 슬롯 = (현재 틱 + 지연) % WHEEL_SIZE`, `회전 수 = 지연 / WHEEL_SIZE`를 계산해
  해당 슬롯에 넣습니다. 만료 위치 계산이 O(1)입니다.
- `tick()`은 커서를 한 칸 전진시키며, 도달한 슬롯에서 회전 수가 0인 타이머를 만료 처리하고
  나머지는 회전 수를 하나 감소시킵니다.
- 한 바퀴보다 긴 지연은 회전 수(rotation)로 처리하는 단일 휠 방식입니다.

### 삭제(remove) 설계 노트

타이머 취소를 O(1)로 처리하기 위해, `add` 시 생성한 노드의 주소를 그대로 식별자로 반환하고
`remove`에서 이를 역참조해 리스트에서 바로 분리하는 방식을 택했습니다.
별도의 `id → 노드` 매핑 테이블을 두지 않아 **메모리·조회 비용을 절약**하는 것이 목적입니다.

대신 이 방식은 다음을 호출자가 보장해야 안전합니다.

- 이미 만료된 타이머의 식별자로 `remove`를 호출하지 않을 것 (해제된 메모리 접근 위험).
- 식별자를 다른 용도로 재사용하지 않을 것.

> 안전성을 우선한다면 정수 id + 매핑 테이블 방식이 대안입니다.
> 현재 구현은 성능을 우선한 선택이며, 트레이드오프를 인지하고 채택했습니다.
